HTML

  • HTLM stand for Hypertext Markup Language.
  • The head element contains information about the webpage.
  • Meta tags provide information about the webpage to the browser.
  • The body element represents the visible content shown to the user.
  • Sections divide site content into parts containing related content.
  • Sections can have class and id attributes which allow for styling and dynamic content.
  • Comments can be used to provide information to the coder. These will not be displayed on the webpage.
  • You can create unordered lists using the ul tag and ordered lists using the ol tag.
  • Each item in a list is contained by the li tag.
  • h1, h2, ..., h6 tags are used for headings of various sizes.
  • Header and footer tags define the content at the top and bottom of the webpage.

CSS

  • CSS stands for Cascading Style Sheets.
  • CSS details how the HTML elements will be styled on the webpage.
  • A CSS rule selects the element to be styled and sets the properties of that section.
  • Each declaration contains a property and value separated by a colon and followed by a semicolon.
  • The universal selector * selects all the elements on the webpage.
  • CSS can be applied through an external stylesheet, internal stylesheet, or inline.
  • A margin indicates how much space we want around the outside of an element
  • Padding indicates how much space we want around the content inside an element
  • Class selectors are defined with a preceeding period in CSS and any HTML element assigned to the class will gain the property assignments from the css

Git

  • git clone [url]: retrieve an entire repository from a hosted location via URL
  • git status: checks what branch we are currently on, shows modified files in working directory
  • git add -A: adds all files from the working branch to the next commit (stage)
  • git reset [file]: unstage a file while retaining the changes in the working directory
  • git commit -m "[descriptive message]": commit your staged content as a new commit snapshot
  • git branch: lists your branches (an astrix will appear next to the currently active branch)
  • git branch [branch-name]: create a new branch at the current commit
  • git checkout [branch-name]: switch to another branch and check it out into your working directory
  • git checkout -b [branch-name]: creates a new branch and switches to it
  • git merge [branch-name]: merges the specified branch's history into the current one
  • git log: shows all commits in the current branch's history
  • git rm [file-name]: delete the file from the project and stage the removal for commit
  • git mv [existing-path] [new-path]: change an existing file path and stage the move
  • git merge [alias]/[branch]: merge a remote branch into your current branch to bring it up to date
  • git push [alias] [branch]: transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch

Javascript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Define a variable: var [variable-name]
  • Functions are defined by the function keyword followed by the function name and parenthesis.
  • The content of a function is encased in curly brackets.
  • Functions are called using the name of the function followed by a set of parentheses: functionName().
  • console.log sends information to the console in the developer tools section of the browser.
  • For loops allow you to automate a task a certain number of times.
  • If statements allow you to execute code only when a condition is met.
